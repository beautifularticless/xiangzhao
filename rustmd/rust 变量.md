rust 变量

## Rust 变量基础

Rust 的变量和其他语言（Java、Python、JS）有 **核心区别** ： **默认不可变** 、 **强类型** 、 **支持可变 / 常量 / 遮蔽** ，这是 Rust 安全特性的基础。

## 1\. 核心：默认不可变（Immutable）

Rust 中声明变量默认 **不能修改值** ，编译会直接报错，强制保证数据安全。

```rust
fn main() {
    // 声明变量：let 关键字
    let x = 5;
    println!("x 的值是：{}", x);

    // 错误！不能修改不可变变量
    // x = 6; 
}
```

✅ 特点：声明后值固定，防止意外修改。

## 2\. 可变变量（Mutable）

需要修改变量时，必须加 **`mut` 关键字** ，显式声明 “这个变量可以改”。

```rust
fn main() {
    // mut = mutable 可变
    let mut y = 10;
    println!("修改前：{}", y);

    // 正确：可变变量可以赋值
    y = 20;
    println!("修改后：{}", y);
}
```

✅ 规则： **只有加了 `mut` ，才能修改变量值** 。

---

## 3\. 常量（Constant）

和不可变变量类似，但 **更严格** ，全程不可变，编译时就确定值。

```rust
fn main() {
    // 常量：const 关键字 + 必须写类型 + 全大写命名
    const MAX_POINTS: u32 = 100_000;
    println!("常量值：{}", MAX_POINTS);
}
```

✅ 常量 vs 不可变变量：

- 常量 **不能用 `mut`** ，永远不可变
- 常量 **必须标注类型**
- 常量可以在 **任何作用域** 声明（包括全局）
- 常量值必须是 **编译期可确定** 的常量表达式

---

## 4\. 变量遮蔽（Shadowing）

Rust 允许 **重复声明同名变量** ，新变量会 “遮蔽” 旧变量， **不是修改值** 。

```rust
fn main() {
    let a = 5;
    // 遮蔽：重新声明同名变量
    let a = a + 1;
    // 遮蔽还可以改变类型！
    let a = "我是字符串了";

    println!("遮蔽后：{}", a);
}
```

✅ 优势：不用创建新变量名，且 **不破坏不可变性** （比 `mut` 更安全）。

---

## 5\. 变量类型

Rust 是 **静态强类型语言** ，编译时必须知道所有变量类型。

### 自动推导类型

```rust
// Rust 自动推导为 i32（整数）
let num = 10;
// 自动推导为 &str（字符串切片）
let str = "hello";
```

### 手动标注类型

```rust
// 格式：变量: 类型
let age: u8 = 18;
let pi: f64 = 3.14159;
let is_ok: bool = true;
```

## 6\. 快速总结对比

表格

| 类型    | 关键字       | 能否修改   | 能否改类型 | 核心特点  |
| ----- | --------- | ------ | ----- | ----- |
| 不可变变量 | `let`     | ❌ 不能   | ❌ 不能  | 默认安全  |
| 可变变量  | `let mut` | ✅ 能    | ❌ 不能  | 显式可变  |
| 常量    | `const`   | ❌ 不能   | ❌ 不能  | 编译期常量 |
| 变量遮蔽  | `let`     | ✅ 逻辑修改 | ✅ 能   | 复用变量名 |

---

### 总结

1. Rust 变量 **默认不可变** ，要修改必须加 `mut`
2. 用 `const` 声明全程不变的常量
3. `let` 重复声明 = 变量遮蔽，可改类型、不破坏安全性
4. 类型可自动推导，也可手动标注
