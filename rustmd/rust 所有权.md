rust 所有权

Rust 的 **所有权（Ownership） **是一套由编译器强制执行的** 编译期规则** ，用于安全高效地管理堆内存，无需垃圾回收（GC）。核心是确保 **每个值仅有一个所有者** ，并在所有者离开作用域时自动释放资源。


### 🔑 核心三规则（必记）

1. **每个值有且仅有一个所有者** （变量）。
2. **同一时刻只能有一个所有者** 。
3. **所有者离开作用域，值被自动丢弃（调用 `drop` ）** 。

---

### 🧠 关键机制详解

#### 1\. 作用域（Scope）

变量从声明点起有效，直到当前作用域（如 `{}` ）结束，此时 Rust 自动调用 `drop` 回收内存。

```rust
fn main() {
    {
        let s = String::from("hello"); // s 进入作用域，有效
        println!("{s}");
    } // s 离开作用域，内存自动释放
    // println!("{s}"); // 编译错误：s 已失效
}
```

#### 2\. 移动（Move）与克隆（Clone）

- **移动（Move）** ：堆数据（如 `String` ）赋值 / 传参时，所有权转移，原变量失效（避免二次释放）。
  
  ```rust
  let s1 = String::from("hello");
  let s2 = s1; // 所有权从 s1 移到 s2，s1 不可再用
  // println!("{s1}"); // 编译错误：值已移动
  println!("{s2}"); // 合法
  ```

- **克隆（Clone）** ：显式深拷贝堆数据，需手动调用 `.clone()` 。
  
  ```rust
  let s1 = String::from("hello");
  let s2 = s1.clone(); // 深拷贝，s1、s2 均有效
  println!("s1={s1}, s2={s2}");
  ```

- **复制（Copy）** ：栈上简单类型（如 `i32` 、 `bool` ）赋值时直接拷贝，不移动，原变量仍可用。
  
  ```rust
  let x = 5;
  let y = x; // 拷贝，x、y 均有效
  println!("x={x}, y={y}");
  ```

#### 3\. 借用（Borrowing）与引用

**临时访问值而不转移所有权** ，通过引用 `&T` （不可变）/ `&mut T` （可变）实现。

- **不可变引用 `&T`** ：允许多个读取者，不可修改。
  
  ```rust
  fn calculate_length(s: &String) -> usize {
      s.len()
  }
  let s = String::from("hello");
  let len = calculate_length(&s); // 借用 s，不转移所有权
  println!("长度：{len}");
  println!("{s}"); // 仍可用
  ```

- **可变引用 `&mut T`** ：同一时间 **只能有一个** 可变引用，且不可与不可变引用共存。
  
  ```rust
  fn change(s: &mut String) {
      s.push_str(", world");
  }
  let mut s = String::from("hello");
  change(&mut s); // 可变借用
  println!("{s}"); // 输出：hello, world
  ```

#### 4\. 生命周期（Lifetime）

**确保引用不会比引用的数据存活更久** ，防止悬垂引用（dangling reference）。

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
// 编译器保证返回的引用与 x、y 生命周期一致
```

### 📊 常见场景速查

表格

| 场景    | 所有权行为                       | 注意事项         |
| ----- | --------------------------- | ------------ |
| 函数传参  | 堆类型（String）移 / 借；栈类型（i32）复制 | 堆类型需避免使用原变量  |
| 函数返回  | 转移所有权                       | 需接收返回值以保留所有权 |
| 多线程共享 | 需用 `Arc` / `Mutex` 等安全原语    | 避免数据竞争       |
| 循环引用  | 需用 `Weak` 引用                | 防止内存泄漏       |

---

### 💡 核心价值

- **内存安全** ：编译期杜绝悬垂指针、二次释放、数据竞争等问题。
- **零开销** ：无运行时 GC 开销，性能接近 C/C++。
- **清晰语义** ：所有权与借用规则明确，代码可维护性高。

---

### 📚 学习建议

1. 从 `String` 、 `Vec` 等堆类型入手，理解移动与复制的区别。
2. 熟练掌握引用规则，尤其是可变引用的 **单一性** 与 **读写互斥** 。
3. 学习生命周期语法，重点解决悬垂引用与函数返回引用问题。
4. 结合 `clone` 、 `Copy` 、 `Arc` 、 `RefCell` 等高级类型，应对复杂所有权场景。
