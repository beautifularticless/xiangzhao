rust 生成文档

## Rust 生成文档完整指南

Rust 内置了 **一流的文档工具 `cargo doc`** ，无需额外安装插件，直接通过注释自动生成美观、可交互的 HTML 文档，还支持文档测试、交叉链接等强大功能。

## 一、快速生成文档（最简用法）

### 1\. 基础命令

在你的 Rust 项目根目录执行：

```bash
# 生成 HTML 文档（默认存放于 target/doc/）
cargo doc

# 生成后**自动打开浏览器**预览
cargo doc --open
```

这就是 Rust 官方标准的文档生成方式，所有 Rust 生态库（如 `tokio` / `serde` ）都用这个命令生成文档。

### 2\. 文档存放位置

生成的文档会放在项目目录：

```
你的项目/target/doc/项目名/index.html
```

直接打开这个 HTML 文件就能查看。

## 二、编写可生成文档的注释（核心）

Rust 有 **专用的文档注释语法** ，普通注释 `//` 不会生成文档，只有以下注释会被解析：

### 1\. 三种文档注释

表格

| 注释语法            | 用途                               |
| --------------- | -------------------------------- |
| `///`           | 为 **下一行** 的项（函数 / 结构体 / 枚举等）生成文档 |
| `//!`           | 为 **当前文件 / 模块** 生成文档（文件顶部说明）     |
| `#[doc = "文字"]` | 宏 / 高级场景用的文档属性（日常不用）             |

### 2\. 完整示例代码

```rust
//! # 我的工具库
//! 这是整个库的顶层文档，写在文件最顶部

/// 计算两个数的和
/// 
/// # 示例
/// \`\`\`
/// let result = my_crate::add(2, 3);
/// assert_eq!(result, 5);
/// \`\`\`
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

/// 用户结构体
/// 
/// 包含用户名和年龄两个字段
pub struct User {
    /// 用户名（字段文档）
    pub name: String,
    /// 年龄
    pub age: u8,
}

impl User {
    /// 创建一个新用户
    pub fn new(name: String, age: u8) -> Self {
        User { name, age }
    }
}
```

执行 `cargo doc --open` 就能生成带标题、代码示例、字段说明的完整文档。

## 三、文档高级用法

### 1\. 文档结构（Markdown 支持）

文档注释 **完全支持 Markdown** ，常用格式：

```rust
/// # 一级标题（文档主标题）
/// ## 二级标题
/// - 列表项
/// \`代码\`
/// **加粗**
/// [链接文字](https://rust-lang.org/)
```

### 2\. 文档测试（可运行的示例）

Rust 最强大的特性： **文档里的代码可以直接编译运行，自动测试正确性** 执行命令运行文档测试：

```bash
cargo test --doc
```

### 3\. 私有项生成文档

默认不生成私有函数 / 结构体的文档，如需包含：

```bash
cargo doc --document-private-items
```

### 4\. 静默生成（不输出日志）

```bash
cargo doc --quiet
```

## 四、常用命令速查表

表格

| 命令                                   | 作用                |
| ------------------------------------ | ----------------- |
| `cargo doc`                          | 生成 HTML 文档        |
| `cargo doc --open`                   | 生成并打开浏览器预览        |
| `cargo test --doc`                   | 运行文档中的测试代码        |
| `cargo doc --document-private-items` | 生成私有项文档           |
| `cargo doc --no-deps`                | 只生成自己的代码，不生成依赖库文档 |

---

## 五、最佳实践

1. 所有 **公有 API** （ `pub` 修饰的函数 / 结构体）必须写 `///` 文档
2. 文档包含： **功能说明 + 参数 + 返回值 + 代码示例**
3. 用 `# 标题` 划分文档结构
4. 用文档测试保证示例代码永远正确

### 总结

1. Rust 用内置工具 `cargo doc` 一键生成专业 HTML 文档
2. 专用注释： `//!`（文件 / 模块）、 `///` （项 / 函数）
3. 支持 Markdown + 可运行文档测试
4. 核心命令： `cargo doc --open` （生成 + 预览）
