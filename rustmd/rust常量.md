## Rust 常量（Constants）

Rust 中的 **常量** 是 **编译期就确定值、全程不可修改、作用域内始终有效** 的固定值，和变量（ `let` ）有本质区别。

## 一、基础语法

```rust
// 格式：const 常量名: 类型 = 值;
const MAX_COUNT: u32 = 100;
const PI: f64 = 3.1415926;
const SERVER_NAME: &str = "rust-server";
```

### 强制规则

1. **必须声明类型** ：常量 **不能省略类型** ，编译器不会自动推导
2. **必须赋值** ：声明时必须直接给定固定值
3. **命名规范** ：全大写 + 下划线（ `MAX_VALUE` ），这是 Rust 约定
4. **不可变** ： **永远不能修改** ，没有 `mut` 写法

## 二、常量 vs 普通变量（let）

表格

| 特性 | 常量（const） | 变量（let） |
| --- | --- | --- |
| 可变性 | 绝对不可变 | 可加 `mut` 可变 |
| 赋值时机 | 编译期确定值 | 运行期可赋值 |
| 类型 | 必须显式声明 | 可自动推导 |
| 作用域 | 全局 / 局部都可用 | 仅局部可用 |
| 内存 | 直接嵌入二进制 | 栈 / 堆分配 |

错误示例（常量不能修改）：

```rust
const A: i32 = 10;
A = 20; // ❌ 编译报错：常量不可赋值
```

## 三、常量的使用场景

1. 固定配置值（端口、最大数量、超时时间）
2. 数学 / 物理常数（π、重力加速度）
3. 全局共享的固定字符串、状态码
4. 数组长度（必须用常量）

```rust
// 数组长度必须用常量
const ARRAY_LEN: usize = 5;
let arr: [i32; ARRAY_LEN] = [1, 2, 3, 4, 5];
```

## 四、全局常量（常用）

常量可以定义在 **函数外、模块顶部** ，作为全局常量，整个文件 / 模块都能使用：

```rust
// 全局常量
const API_VERSION: &str = "v1.0.0";
const DB_PORT: u16 = 3306;

fn main() {
    println!("API: {}", API_VERSION);
    println!("端口: {}", DB_PORT);
}
```

## 五、编译期计算常量

Rust 支持 **编译期计算** ，常量可以用简单表达式赋值，运行时无开销：

```rust
const TWO_HOURS: u32 = 60 * 60 * 2; // 7200，编译期直接算出结果
const AREA: f64 = PI * 5.0 * 5.0;
```

## 六、静态变量（static）：特殊的 “全局常量”

和常量很像，但 **有固定内存地址** ，支持可变（不安全），全局生命周期：

```rust
// 不可变静态变量（和常量类似，更强调全局唯一地址）
static LANGUAGE: &str = "Rust";

// 可变静态变量（必须用 unsafe，不推荐新手使用）
static mut COUNT: i32 = 0;
```

**日常优先用 `const`** ， `static` 只在需要全局内存地址时使用。


### 总结

1. **`const NAME: Type = value;`** 是 Rust 常量标准写法
2. 核心特性： **编译期确定、绝对不可变、必须写类型**
3. 用于固定值、全局配置、数组长度等场景
4. 优先用 `const` ，少用 `static`
