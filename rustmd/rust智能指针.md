rust智能指针

Rust 智能指针（Smart Pointer）是封装了裸指针、并附加所有权 / 生命周期 / 引用计数 / 线程安全等逻辑的结构体，核心是在 **安全、无 GC** 前提下灵活管理堆内存与共享所有权。下面从核心类型、关键 Trait、组合用法与选型四个方面系统讲解。

---

### 一、核心智能指针一览

#### 1\. Box<T>：唯一所有权・堆分配

- **原理** ：栈存指针、堆存数据； **单一所有权** ，离开作用域自动释放。
- **关键 Trait** ：Deref（自动解引用）、Drop（自动释放）。
- **适用场景** ：
	- 递归类型（如链表 / 树，解决 “无限大小” 问题）；
		- 大数据转移（避免栈拷贝）；
		- Trait 对象（dyn Trait，动态分发）。
- **示例** ：

```rust
fn main() {
    let b = Box::new(5); // 堆上存i32，栈存指针
    println!("{}", *b); // 自动解引用
} // b离开作用域，堆内存自动释放
```

#### 2\. Rc<T>：单线程・引用计数共享

- **原理** ： **引用计数（共享所有权）** ，clone () 计数 + 1，drop () 计数 - 1；计数为 0 时释放。
- **关键限制** ： **单线程专用** 、内部默认 **不可变** 。
- **适用场景** ：单线程内多所有者只读共享（如图节点、配置）。
- **示例** ：

```rust
use std::rc::Rc;
fn main() {
    let a = Rc::new(10);
    let b = Rc::clone(&a); // 计数=2
    println!("count: {}", Rc::strong_count(&a));
} // 计数降为0，释放
```

#### 3\. Arc<T>：多线程・原子引用计数

- **原理** ：Rc 的 **线程安全版** ，用 **原子操作** 维护引用计数，可跨线程共享。
- **适用场景** ：多线程间只读共享数据（全局配置、缓存）。
- **示例** ：

```rust
use std::sync::Arc;
use std::thread;
fn main() {
    let a = Arc::new(20);
    let h = thread::spawn({
        let a = Arc::clone(&a);
        move || println!("{}", a)
    });
    h.join().unwrap();
}
```

#### 4\. RefCell<T>：单线程・内部可变性

- **原理** ： **运行时借用检查** （替代编译期），实现 “不可变容器内可变数据”。
- **关键限制** ： **单线程专用** ；运行时违反借用规则会 panic。
- **方法** ：borrow ()（不可变借用）、borrow\_mut ()（可变借用）。
- **示例** ：

```rust
use std::cell::RefCell;
fn main() {
    let x = RefCell::new(5);
    *x.borrow_mut() += 1; // 可变借用
    println!("{}", x.borrow()); // 不可变借用
}
```

#### 5\. 组合：Rc<RefCell<T>> / Arc<Mutex<T>>

- **Rc<RefCell<T>>** ：单线程 **共享 + 可变** （如树节点互改）。

```rust
type Shared<T> = Rc<RefCell<T>>;
```

- **Arc<Mutex<T>>** ：多线程 **共享 + 可变** （线程安全）。

```rust
type ArcMut<T> = Arc<Mutex<T>>;
```

#### 6\. Weak<T>：弱引用・解决循环引用

- **原理** ：Rc/Arc 的 **弱引用** ，不增加强计数；强计数为 0 时弱引用自动失效， **避免循环引用泄漏** 。
- **示例** ：

```rust
use std::rc::{Rc, Weak};
struct Node { val: i32, parent: Weak<Node> }
```

#### 7\. 其他常用

- **Cell<T>** ：Copy 类型的内部可变性（get/set，无借用检查）；
- **Cow<'a, T>** ：写时克隆（只读时借用，修改时克隆）；
- **Pin<T>** ：固定内存位置（防止自引用结构体移动）。

---

### 二、核心 Trait：Deref 与 Drop

1. **Deref** ：实现 `deref(&self) -> &T` ，支持 `*` 解引用与 **自动强制转换** （&Box<T> → &T）。
2. **Drop** ：实现 `drop(&mut self)` ，离开作用域时 **自动调用** ，释放资源。

---

### 三、智能指针选型速查表

表格

| 类型 | 所有权 | 线程安全 | 可变性 | 核心场景 |
| --- | --- | --- | --- | --- |
| Box<T> | 唯一 | 无关 | 直接可变 | 递归类型、大数据、Trait 对象 |
| Rc<T> | 共享 | ❌ | 需 RefCell | 单线程只读共享 |
| Arc<T> | 共享 | ✅ | 需 Mutex | 多线程只读共享 |
| RefCell<T> | 唯一 | ❌ | 内部可变 | 单线程内部可变性 |
| Weak<T> | 弱共享 | ✅(Arc)/❌(Rc) | 无 | 打破循环引用 |

---

### 四、常见坑与最佳实践

1. **Rc/Arc 循环引用** ：用 \*\*Weak<T>\*\* 打破（如父节点存弱引用子节点）；
2. **RefCell 双重可变借用** ：运行时 panic，需避免同一作用域多个 borrow\_mut ()；
3. **Arc<RefCell<T>>** ： **不安全** ！跨线程用 **Arc<Mutex<T>>** ；
4. **优先 Box** ：默认堆分配用 Box，简单高效。
