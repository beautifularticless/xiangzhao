rust生命周期

### 一、生命周期是什么

**生命周期（Lifetime）** 是 Rust 用来在 **编译期** 保证 **引用永远有效** 的一套标记系统，核心目的是 **杜绝悬垂引用（dangling reference）** 。

- 记法：用 **撇号 + 小写标识符** 表示，如 `'a` 、 `'b` 、 `'ctx` 。
- 本质：描述 **引用的有效范围** ，是 **编译期概念，无运行时开销** 。
- 核心约束： **引用的生命周期 ≤ 被引用数据的生命周期** 。

### 二、为什么需要生命周期（悬垂引用案例）

下面代码在 Rust 中 **直接编译失败** ，因为 `r` 指向的 `x` 已经销毁：

```rust
fn main() {
    let r;
    {
        let x = 5;
        r = &x; // x 的生命周期到此结束
    }
    println!("{}", r); // 错误：r 是悬垂引用
}
```

生命周期就是编译器用来 **静态阻止这类错误** 的工具。

### 三、基础语法：函数中的生命周期标注

当 **函数返回引用** 时，编译器需要知道： **返回值的引用来自哪个参数** 。

#### 1\. 错误示例（无法编译）

```rust
fn longest(x: &str, y: &str) -> &str { // 编译器不知道返回值和x/y的关系
    if x.len() > y.len() { x } else { y }
}
```

#### 2\. 正确标注（'a 绑定输入输出生命周期）

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

含义：

- `'a` 是 **生命周期参数** 。
- 所有输入参数和返回值 **共享同一个生命周期 `'a`** 。
- 保证： **返回的引用至少和两个输入引用活得一样长** 。

### 四、生命周期省略规则（Lifetime Elision）

Rust 有 **三条省略规则** ，让 **90% 场景无需手写生命周期** ：

1. **每个输入引用参数 → 独立生命周期**
	- `fn foo(x: &str)` → `fn foo<'a>(x: &'a str)`
2. **只有一个输入生命周期 → 输出绑定到该生命周期**
	- `fn foo(x: &str) -> &str` → `fn foo<'a>(x: &'a str) -> &'a str`
3. **方法中 `&self` / `&mut self` → 输出绑定到 `self` 生命周期**
	- `impl S { fn get(&self) -> &T }` → 自动绑定

### 五、结构体中的生命周期

**结构体包含引用时，必须显式标注生命周期** ，确保 **引用的数据比结构体实例活得久** 。

```rust
// 结构体 Book 包含一个引用字段 title
struct Book<'a> {
    title: &'a str,
}

fn main() {
    let title = String::from("Rust 编程");
    let book = Book { title: &title }; // 合法：title 比 book 活得久
}
```

### 六、特殊生命周期：'static

- **`'static`** ：表示 **整个程序运行期都有效** 的生命周期Rust。
- 常见场景： **字符串字面量** （硬编码在程序二进制中）。

```rust
let s: &'static str = "hello"; // 全局有效
```

### 七、生命周期约束：'a: 'b

- 读作： **`'a` 活得比 `'b` 久（或一样久）** 。
- 用途：约束两个生命周期的长短关系。

```rust
fn longer<'a, 'b: 'a>(x: &'a str, y: &'b str) -> &'b str {
    y
}
```

### 八、常见错误与解决

1. **“missing lifetime specifier”**
	- 原因：返回引用但未标注生命周期。
		- 解决：按规则添加 `'a` 绑定输入输出。
2. **“lifetime `'a` does not live long enough”**
	- 原因：引用的数据提前销毁。
		- 解决： **延长数据生命周期** 或 **缩短引用生命周期** 。

### 九、核心总结

- 生命周期是 **编译期引用有效性检查** ， **无运行时成本** 。
- 核心目标： **防止悬垂引用** ，保证 **引用始终指向有效内存** 。
- 多数场景可 **自动省略** ，只需在 **函数返回引用、结构体含引用** 时手动标注。
